```bash

  "username": "testuser",
  "password": "SecurePass123!@#",
  "email": "test@example.com",
  "first_name": "Test",
  "last_name": "User"

# 2. Login
POST http://localhost:8000/api/auth/login/
{
  "username": "testuser",
  "password": "SecurePass123!@#"
}

# 3. Get current user (requires token)
GET http://localhost:8000/api/auth/me/
Header: Authorization: Bearer {access_token}

# 4. Get student schedule (requires token)
GET http://localhost:8000/api/student/schedule/
Header: Authorization: Bearer {access_token}

# 5. Get admin dashboard (requires token, all users can access)
GET http://localhost:8000/api/admin/master-pulse/
Header: Authorization: Bearer {access_token}
```

## 🎓 How It Works

### Registration Flow
```
User fills form → Backend validates → Creates User → Creates UserProfile → Creates Student → Success
```

### Login Flow
```
User enters credentials → Backend validates → Returns tokens → User stores tokens → Can access API
```

### API Access Flow
```
User makes request + token → Backend validates token → Token valid? → YES → Return data
                                                                    → NO → 401 Unauthorized
```
